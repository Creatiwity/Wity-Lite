# Wity Changelog

0.3.0 changes
-------------





v0.2
----

WTemplate
- Parser recodé (réglage de nombreux bugs)
- Gestion des short tags (<?xml)
- Correction d'un bug avec les chemins de compilation qui contiennent des caractères "."
- Fixed a bug when a tag {} was present at the begining of a file
- New nodes: {for $key, $value in $array} and {empty} in case of empty $array
- Modified node: {include href}
- Deleted node {block} and method WTemplate::assignBlockVars no longer exists
- The specified cache directory is automaticly created if no exists

WMain
- Template configuration déplacée directement dans WView

WRoute
- Correction d'un bug de routage lorsque wity se trouve à la racine

WView
- Special theme "_blank" added
- Method "setTpl" renamed into "setResponse"
- New method "findResponse" moved from WController::findTpl()
- Handles WTemplate's exceptions while compiling

Apps
- User : correction du bug de redirection à la connexion

WNote
- Class moved to %SYS_DIR%/WCore/
- WException removed
- Removed handler session: all notes are saved into session and are automaticly treated by WView
- New handlers 
- New method "display_full" handling a set of notes. It displays them in a full custom error page

WSession
- Now handles error while trying to connect to the database

WLang
- Creation of WLang
- Linked with WTemplate : {lang id}default{/lang} will find the translation for the entry "id" 
  with the failover default string or use directly {lang}id{/lang}

Apache
- Force to use UTF-8 by default in .htaccess
